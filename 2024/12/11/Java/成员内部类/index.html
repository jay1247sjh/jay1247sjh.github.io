成员内部类

发表于2024-12-11|更新于2024-12-12|Java
|总字数:710|阅读时长:2分钟|浏览量:

什么是内部类

内部类的特点

内部类可以直接访问外部类的属性，外部类访问内部类的时候必须创建对象

内部类的总结

  • 内部类写在外部类里面，可以被一些修饰符所修饰，比如private，默认，protected，public，static（静态内部类）

  • 在成员内部类里面，JDK-16之前不能定义静态变量，JDK-16之后才可以定义

  • 加了权限修饰符之后，访问权限跟成员变量加了权限修饰符的效果是一样的

外部无关类直接创建内部类对象

通过外部类的函数进而创建内部类

要给内部类加上static才能使用Car.Engine创建对象

当内部类使用了static的时候，就不能直接使用外部类的成员对象了

内部类加上static就报红

但假如内部类的访问权限为private，我们就无法在外部无关类中运用Car.Engine类型来接收对象的地址，可以通过多态（即直接Car类型接收）或是创建对象后直接访问属性

局部内部类

匿名内部类的写法

实际上，以上写法更像是创建对象，类的是从@Override前面的 { 开始到最后 } 结束才是匿名内部类，前面不过是创建了一个对象

反编译之后可以看到Java默认加上名字叫Test$1，并且继承了Swim接口

new 一个类的话则是继承关系，new一个接口的话则是实现关系，需要在类中重写所有抽象方法

匿名内部类的具体作用

在学习匿名内部类之前，我们想要调用Show方法的时候，首先要创建一个类并且实现Swim，然后再创建该实现类的对象去传参，显得十分繁琐，而匿名内部类则很好地解决了这个问题，创建匿名对象并且实现了接口的抽象函数，并且可以直接传参，实现了接口的多态效果

匿名内部类还可以这样用

直接写在成员位置也可以
