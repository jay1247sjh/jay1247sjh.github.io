static的注意事项

发表于2024-12-11|更新于2024-12-12|Java
|总字数:384|阅读时长:1分钟|浏览量:
  • 静态方法只能访问静态变量和静态方法

主函数访问非静态方法

  • 非静态方法可以访问静态变量或是静态方法，也可以访问非静态的成员变量和非静态的成员方法（具有隐藏的this）

隐藏了一个this，虚拟机自动赋值，无法手动赋值（表示当前方法调用者的地址）

this的调用（实际上为this.name/this.age）

非静态方法可以访问静态方法和非静态方法

当s1调用show1时，在show1调用show2的使用也是使用s1的地址。s2同理

  • 静态方法当中没有this关键字（静态方法不能调用实例变量）

Java的方法区（Method Area）是一块用来存储类信息、常量、静态变量、即时编译器编译后的代码等数据的内存区域

内存图（JDK-7后才有单独的静态区）

在编译的时候，字节码文件会放在方法区，当创建了静态变量的时候，会放在堆区的静态区，当调用method打印时，因为是static方法，所有会直接在静态区找变量，但静态区显然不包含name，所以会报错

当静态区的值初始化完成之后，才叫做字节码文件加载成功

说明非静态的可以访问所有的东西，图中先创建了一个Student的对象，然后根据对象在方法区寻找静态方法
