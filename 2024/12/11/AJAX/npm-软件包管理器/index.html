npm-软件包管理器

发表于2024-12-11|更新于2024-12-12|AJAX
|总字数:217|阅读时长:1分钟|浏览量:

固化软件包的版本目的是为了防止原作者发布新版本对于里面的一些方法进行改进对于原来使用方法的代码带来的影响

npm init -y 初始化清单文件，得到package.json文件，-y表示默认值

通过package.json来管理项目依赖项、运行脚本、发布包等。此外，很多 Node.js 工具和框架也会根据 package.json 文件中的信息来进行配置和初始化

npm i dayjs 导入dayjs包

node_modules一般直接npm i导入而不是拷贝

在06这个文件夹的package.json文件中记录了依赖包的名字和版本，但是缺少了node_modules文件夹

npm i 命令创建node_modules文件夹，并导入相关包
