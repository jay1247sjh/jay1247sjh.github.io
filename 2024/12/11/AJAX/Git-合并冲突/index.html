Git-回退版本、删除文件
**git reset –soft 版本号 ** 使用git reset –soft之后，不会删除相对于版本库中存储文件之后新增的文件，只不过会把这些文件变为U未跟踪状态，一般适用于恢复已经存储过的文件而不影响新增文件的情况 git reset –soft 也会保留缓冲区的文件 **git reset –mixed 版本号 ** 使用git reset –mixed之后，工作区的文件还是会保留，在存储之后新增的文件也是变成U状态，但唯一的不同在于缓冲区的内容被清空了 git reset –hard 版本号 按理来说，为什么执行了git reset –hard之后，相对于index.html之外新增的文件还保留在工作区呢？这是因为git reset –hard的工作原理是：先把对应版本的内容恢复到暂存区，然后对比暂存区与工作区的内容，随后进行恢复。但由于我们之前的操作已经把暂存区给恢复了，所以这里没有去对比和删除其他文件 可以先 git add. 把文件存到暂存区，然后 git reset –hard，这样就可以恢复了 git reflog –oneline...
