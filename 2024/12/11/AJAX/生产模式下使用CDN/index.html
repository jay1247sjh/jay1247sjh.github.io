生产模式下使用CDN

发表于2024-12-11|更新于2024-12-12|AJAX
|总字数:548|阅读时长:1分钟|浏览量:

利用CDN访问第三方库，可以不用把第三方库的源码一同打包，从而提升效率

免费CDN第三方库

首先在想要使用CDN库的文件用 模板 + 导入 的方式写入链接

为什么要用模板呢？因为这里的目的是，在开发环境下，我们用直接导入的方式；在生产环境下，我们用CDN。所以这个模板的作用是用来判断是否为生产环境的，是则引入CDN的库

options是一个变量或对象，用于存储配置选项或设置。在这段代码中，htmlWebpackPlugin是一个插件对象，它应该定义了一个options属性，用于存储用户定义的配置选项。通过读取htmlWebpackPlugin.options.useCdn属性，代码可以根据用户的配置动态地加载CSS资源或不加载

useCdn是自己起的变量名

这里是忽略打包的配置，为了在生产环境下不导入这个库，而是导入CDN的库

这里的键写的是import后面的库名，值写的是留在原地的全局变量，我猜测这里留下全局变量的意思是为了让全局变量接收CDN导入过来的库

最后在plugins这一栏把自己的自定义属性useCdn填上去，判断是否为production环境，如果为true则执行模板命令，也就是导入CDN

查看一下打包和开发的状态（–mode）

当我在开发环境下 npm run dev，发现是没有CDN地址的

可以看到开发模式下的打包是没有把CSS打包进来的

总结:引入模板 -> 引入外部扩展 -> 添加插件
