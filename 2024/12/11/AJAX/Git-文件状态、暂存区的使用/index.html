Git-文件状态、暂存区的使用

发表于2024-12-11|更新于2024-12-12|AJAX
|总字数:266|阅读时长:1分钟|浏览量:

当我新添加一个文件的时候是U状态（未跟踪）

当用 git add . 命令把新文件放入缓存区，状态变为A

使用 git status -s 查看暂存区文件状态（-s可以去掉，其表示更简介的内容）

当我修改了一下CSS代码，发现工作区的状态变为M了

在工作区状态未M的情况下更新一下缓存区，状态变为A

当把缓存区的内容上传到版本库中，文件右侧的状态消失了（实际上是空字符串代表未修改的意思）

再次修改CSS代码，M重新出现（恢复到版本库的状态的时候M会消失）

暂存区移除文件之后，变为未跟踪U状态

恢复前

**git restore 文件名 恢复后，可以看到自动覆盖了原来的内容 **

git rm –cached 文件名 进行缓存区的删除文件操作，删除后可以看到状态变为U了

git log –oneline 查看所有comit的历史
