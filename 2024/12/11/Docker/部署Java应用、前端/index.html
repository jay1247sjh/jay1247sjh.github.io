部署Java应用、前端

发表于2024-12-11|更新于2024-12-12|Docker
|总字数:219|阅读时长:1分钟|浏览量:

后端

将写好的Java应用通过Maven进行打包，并放到虚拟机中

将传入的jar包转化为镜像，并命名为hmall

根据镜像创建名为hm的容器并启动，然后根据虚拟机ip+端口号（这里是8080）就可以访问了

前端

将文件上传到虚拟机，然后进行创建容器并跑起来

nginx配置文件要写对代理的后台容器，不然没有办法正确启动服务

因为前端跑起来的时候，需要把html文件放入nginx的html文件夹，所以需要挂载。而后端的直接用jar包生成镜像再创建容器运行就可以了

创建nginx容器的时候要指定网络，并且与后端是同一个，否则没办法连的上

虚拟机ip+映射端口号，访问成功
